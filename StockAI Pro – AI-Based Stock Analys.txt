StockAI Pro – AI-Based Stock Analysis & Decision Support System

📄 Abstract

Stock market investment has always been a complex and high-risk domain, requiring investors to process large volumes of financial data, news, and market trends simultaneously. Traditional stock analysis tools often lack intelligent decision support, leaving retail investors without proper guidance. This project presents StockAI Pro, an AI-powered web-based stock analysis and decision support system designed to bridge this gap by combining real-time market data, machine learning predictions, and natural language AI assistance into a single unified platform.
The system is developed using Django as the backend framework, PostgreSQL as the relational database, and HTML, CSS, JavaScript, and Bootstrap for a responsive and intuitive frontend interface. Real-time stock data is fetched through industry-standard financial APIs including Yahoo Finance and Alpha Vantage, ensuring data accuracy and market reliability. The platform strictly follows Indian stock market timing — 9:15 AM to 3:30 PM IST — displaying live data during market hours and historical data after closing.
StockAI Pro consists of six core modules. The Stock Dashboard Module provides live stock prices, interactive candlestick and line charts, and technical indicators such as RSI (Relative Strength Index) and Moving Averages, along with a personalized watchlist feature. The AI Prediction Module employs machine learning techniques, beginning with Linear Regression and scalable to Long Short-Term Memory (LSTM) neural networks, to forecast stock price trends and classify them as bullish or bearish with an associated confidence score. The Sentiment Analysis Module fetches real-time financial news and applies natural language processing to determine positive or negative market sentiment, linking it directly to corresponding stock movements.
The Portfolio Management Module enables users to add and track their stock holdings, monitor real-time profit and loss, and receive AI-driven investment suggestions. The Alerts System notifies users of significant price movements and AI-detected anomalies in market behavior. Finally, the AI Chat Assistant Module, powered by a large language model API, provides intelligent responses strictly limited to stock market related queries. A robust prompt filtering and guardrail mechanism ensures the assistant declines all unrelated questions with a polite, predefined response — making it a focused and reliable financial assistant.
StockAI Pro is designed as a modular, scalable, and production-ready system that demonstrates the practical integration of artificial intelligence, real-time data processing, and modern web technologies in the financial domain. The project aims to empower retail investors with intelligent tools previously available only to institutional players, making informed stock market decisions accessible to everyone.

# StockAI Pro — Complete System Documentation

---

## 1. PROJECT OVERVIEW

**Project Name:** StockAI Pro — AI-Based Stock Analysis & Decision Support System

**Objective:** A full-stack web application that provides real-time Indian stock market data, AI-powered predictions, sentiment analysis, portfolio tracking, smart alerts, and an intelligent stock-only AI chat assistant.

**Problem It Solves:** Retail investors lack access to intelligent, unified tools for stock analysis. StockAI Pro combines real-time data, ML predictions, news sentiment, and AI chat in one platform — previously available only to institutional players.

**Target Users:** Indian retail stock market investors, finance students, traders

**Core Features:**
1. Live Stock Dashboard (NSE/BSE real-time data, RSI, MA20, charts)
2. AI Price Prediction (Linear Regression + Neural Network/LSTM)
3. News Sentiment Analysis (TextBlob NLP on real news)
4. Portfolio Tracker (holdings, P&L, transaction history)
5. Smart Price Alerts (above/below threshold triggers)
6. AI Chat Assistant (Groq AI, stock-only guardrail)
7. User Authentication (register, login, logout, profile)
8. Landing Page (public-facing marketing page)

```
You are StockAI Pro Assistant, an expert stock market analyst for Indian markets (NSE/BSE).
You ONLY answer questions related to:
- Stock markets (NSE, BSE, Sensex, Nifty)
- Individual stocks and companies
- Trading strategies and techniques

- Financial metrics (PE ratio, EPS, RSI, etc.)
- Market news and analysis
- Mutual funds and ETFs
- Indian economy and RBI policies
Keep responses concise, clear and actionable.
Always mention risks when giving investment advice.
Format numbers in Indian style (Lakhs, Crores).
```

**Guardrail Logic:**
```python
# Step 1: Check if greeting → allow
# Step 2: Check if any STOCK_KEYWORD in message → allow
# Step 3: Otherwise → block with:
"I am designed to answer only stock market related queries.
Please ask me about stocks, trading, investments, or financial markets!"
```

**Response Handling:**
- Allowed → Groq API called → response.choices[0].message.content returned
- Blocked → is_blocked=True, canned response returned
- Both saved to ChatLog DB
- Frontend shows blocked in red bubble, allowed in normal AI bubble

## 9. DATA FLOW ARCHITECTURE

```
USER ACTION → FRONTEND → BACKEND → API → RESPONSE → UI

1. Dashboard Load:
   User visits /dashboard/
   → dashboard_view()
   → get_multiple_stocks(['RELIANCE','TCS',...])
   → yf.Ticker("RELIANCE.NS").info + .history()
   → calculate_rsi(prices) + calculate_ma(prices)
   → json.dumps(chart_data)
   → render dashboard.html with context
   → Chart.js renders price + RSI charts

2. AI Chat:
   User types message → JS fetch POST /chat/api/
   → chat_api view
   → is_stock_related(message) check
   → If YES: Groq API call with system prompt
   → response.choices[0].message.content
   → ChatLog.objects.create(...)
   → JsonResponse({'response': text, 'is_blocked': False})
   → JS appends bubble to chat

3. Portfolio P&L:
   User visits /portfolio/
   → portfolio_view()
   → Portfolio.objects.filter(user=request.user)
   → For each holding: get_stock_data(symbol)
   → current_value = quantity × current_price
   → pnl = current_value - total_invested
   → render portfolio.html

4. Sentiment:
   User visits /sentiment/?symbol=TCS
   → sentiment_view()
   → fetch_stock_news('TCS', 'Tata Consultancy...')
   → NewsAPI /v2/everything → 10 articles
   → For each: TextBlob(title+description).sentiment.polarity
   → Average polarity → overall sentiment
   → render sentiment.html with doughnut chart
```

## 12. REAL-TIME DATA HANDLING

**Stock Data Source:** Yahoo Finance via yfinance library

**Fetch Trigger:** Every page load (no auto-refresh, manual refresh)

**Market Timing Logic:**
```python
def is_market_open():
    ist = pytz.timezone('Asia/Kolkata')
    now = datetime.now(ist)
    open_time = now.replace(hour=9, minute=15, second=0)
    close_time = now.replace(hour=15, minute=30, second=0)
    is_weekday = now.weekday() < 5  # Mon-Fri
    return is_weekday and open_time <= now <= close_time
```

**Dashboard behavior:**
- Shows "Market Open" green badge during 9:15AM-3:30PM IST weekdays
- Shows "Market Closed" red badge otherwise
- Data still loads regardless (shows last available price)
- Real-time clock in dashboard updates every second via JS `setInterval`

**Data freshness:** yfinance fetches latest available data from Yahoo servers on each page load. During market hours this is near real-time (~15 min delay).

## 15. COMMON RISKS / BREAKPOINTS

### What Can Break:
1. **yfinance API changes** → stock data returns None → dashboard crashes
   - Fix: Always wrap in try/except, return None, check in view
2. **Groq API key quota** → chat returns error
   - Fix: Check quota at console.groq.com, key in .env
3. **NewsAPI rate limit** → 100 requests/day free tier
   - Fix: Don't call on every refresh; consider caching
4. **Django template `{% endblock %}` missing** → TemplateSyntaxError
   - Fix: Always verify block tags are matched
5. **`apps/` folder `__init__.py` missing** → ModuleNotFoundError
   - Fix: Ensure `apps/__init__.py` exists
6. **PostgreSQL not running** → connection error on startup
   - Fix: Start PostgreSQL service before `runserver`
7. **`.env` not loaded** → API keys return None
   - Fix: `load_dotenv()` must be at top of `settings.py`
8. **TATAMOTORS vs TATAMOTORS.NS** → yfinance may fail
   - Fix: `.NS` suffix added in `api_service.py`

### What to NEVER Do:
- Never delete `apps/__init__.py`
- Never hardcode API keys in Python files
- Never change `name` in `apps.py` without updating `settings.py`
- Never add `{% endblock %}` in wrong position
- Never use Bootstrap dark theme classes — custom dark theme only
